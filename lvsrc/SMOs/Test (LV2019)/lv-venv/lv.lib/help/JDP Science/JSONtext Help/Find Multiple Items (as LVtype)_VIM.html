JSONtext.lvlib:Find Multiple Items (as LVtype).vim

Find the items at the JSON Path specified, converts them to a LabVIEW datatype, and returns them as an array.

Paths specify an item in a tree of nested Objects, and start with $. Paths may point to multiple items, in the presence either of Arrays, or of multiple specified locations (using a [] form, see example).

Example JSON:

{"A":[{"C":123,"D":4},{"C":567,"D":8}],"B":false,"E":true}

At path $.A.C are 123 and 567 (which can be returns as an array of integers)

At path $.[B,E] are false and true (which can be returned as an array of booleans)

 JSON

Input JSON. Optional "Offset" indicates an offset in this string pointing to the start of the JSON item to operate on.

$Path or Item Name

JSON Path notation starts with a $. If not starting with a $, this input is treated as an item name (so <name> and $.<name> are equivalent).

Offset

Optional Offset in the input "JSON" string. Default is zero, the start of the string. This Offset should point to the start of the JSON Value of interest (or to whitespace in front of it).

error in (no error)

Describes error conditions that occur before this node runs. This input provides standard error in functionality.

    		status

    		code

    		source

Data Type and Default Value

Data type you would like to convert the JSON to. This input accepts most LabVIEW types, including arrays and clusters of those types. The input also sets the default value to return if it is not possible to convert the JSON (due to a missing item or type mismatch).

LVClass Serializer

Optional way to specify a JSON format for LabVIEW Objects. Provide a Child Class at this input that impliments LVObject<-->JSON conversion.

If no Serializer is provided, then LVObjects are serialized by the default method (flattened object in a JSON string).

error out

Contains error information. This output provides standard error out functionality.

    		status

    		code

    		source

Names

    		result string

Item Offsets

Offsets, in the input JSON, of the found items.

    		Item offset
