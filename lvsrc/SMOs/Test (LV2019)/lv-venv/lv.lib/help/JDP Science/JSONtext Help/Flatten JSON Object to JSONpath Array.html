JSONtext.lvlib:Flatten JSON Object to JSONpath Array.vi

Takes a JSON Value and "flattens" the Object structure, producing out arrays of the $Paths to each item and the corresponding items. This works recursively on all subObjects (but not Arrays, which are treated as a single item).

Note: to support names containing periods (.) in them, those names will be returned as a JSON string (quoted).

For example: the JSON {"A":{"B1":1,"B2":"x"},"A2":true,"A.1":{"C":123}} produces the arrays:

$Paths (Items):

$.A.B1 (1)

$.A.B2 ("x")

$.A2 (true)

$."A.1".C (123)

If "Include $Paths to Objects" is True, then the above list will incude a path to all Objects (including the root Object path "$"):

$

$.A

$.A.B1

$.A.B2

$.A2

$."A.1"

$."A.1".C

 JSON

Input JSON. Optional "Offset" indicates an offset in this string pointing to the start of the JSON item to operate on.

Offset

Optional Offset in the input "JSON" string. Default is zero, the start of the string. This Offset should point to the start of the JSON Value of interest (or to whitespace in front of it).

Initial Path

Optional initial path added to the front of the Paths. The default is just '$', the standard start of a JSON Path.

error in (no error)

Describes error conditions that occur before this node runs. This input provides standard error in functionality.

    		status

    		code

    		source

Maximum Depth

Number of Levels of subclusters to Flatten. Default -1 means Flatten all levels.

Include $Paths to Objects

Include $Paths to Objects, in addition to the Objects' subitems.

$Paths

Array of JSON Paths.

    		Path

<JSON>Items

Output JSON. The <JSON> tag indicates to some downstream functions that this is JSON, rather than a standard LabVIEW string.

    		<JSON>

error out

Contains error information. This output provides standard error out functionality.

    		status

    		code

    		source
