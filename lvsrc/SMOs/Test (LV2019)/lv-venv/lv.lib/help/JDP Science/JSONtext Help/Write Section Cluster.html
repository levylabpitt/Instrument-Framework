JSONtext.lvlib:Write Section Cluster.vi

Merges a Section Cluster into JSON Object. A "Section Cluster" is defined in analogy to the OpenG Variant Config Library "Section Cluster" functions, using a JSON Object, with subObjects representing "Sections", instead of INI format.

Merging means that each element of the Section Cluster is inserted into the JSON "Section" (Object Item), using the data name of the cluster element. If a Path to the Section is not specified then the data name of the cluster is used. The Section is created if it does not exist.

For example, Merging the Cluster "Settings" with elements A=123 and B="test" into {"C":2,"Settings":{"A"=6,"D":9}} would result in:

{"C":2,"Settings":{"A"=123,"D":9,"B":"test"}}

Note that this merging is not recursive into subClusters.
