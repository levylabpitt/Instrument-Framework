Offsets in JSONtext

Offsets are optional.  They can be used to improve performance, such as by finding an item, changing it, and updating it in the original JSON without the overhead of finding the item twice.  In general, however, the library User should ignore offsets all together.

An "offset" refers to a location in the input "JSON" string at which the actual JSON Value of interest starts (or to whitespace in front of that start).   

For example, in the JSON {"A":1,"B":[2,3]}, offset=5 is 1, while offset=10 is [2,3] and offset=11 is 2.

For support please see [CR] JSONtext.
