LabVIEW Class Support

By Default, Objects of LabVIEW Classes are converted to the standard flattened binary form of the LabVIEW Class, stored in a JSON string.  However, the functions that convert to/from LabVIEW type have an optional input of a "LVClass Serializer".  This is itself a LabVIEW class that has two overridable methods for conversion of LVObjects to or from JSON.  Custom JSON formating of LabVIEW Objects can thus be provided via a Child Serializer Class.

For more on Class Support, please see Bitbucket Issue #56.

For support please see [CR] JSONtext.
