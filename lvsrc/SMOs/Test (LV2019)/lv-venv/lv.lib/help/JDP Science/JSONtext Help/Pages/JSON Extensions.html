Extensions to JSON supported by JSONtext

Beyond the standard JSON grammer, JSONtext adds the following support:

By default, support the "LabVIEW Extension" of NaN, Infinity and -Infinity values (in strict JSON, these are all null).   This can be disabled with an input on JSONtext functions that convert from LabVIEW types.  Alternately, using the Conditional Symbol "JSONtext_Strict==True" will force all such values to be converted to null.  Note that JSONtext will always accept these spacial values, even when it will not write them.

Comments: supports comments of marked by

  • //My Comment 
  • /*my inline comment*/

Special comma treatment in lists (Objects of Arrays), meant to recover from common User editing mistakes:

  • If a comma in an Object or Array is missing, but there is a new line, then a comma is assumed.
  • If there is a trailing comma at the end of a list then that comma is ignored.

(see the example "Comments and Commas.vi").

Example JSON:

// Supports Comments
{
"a":1 // like this
"b":2 /*or this style
of multiline comment*/
"c": 3 //oh, and notice I'm forgetting some commas
"d":4, // except I've left a trailing one at the end
}
