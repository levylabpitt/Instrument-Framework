JSONtext.lvlib:Not Equivalent.vi

Returns True if two JSON Values are "Not Equivalent".

Equal means they are the same JSON infomation. Not considered differences are:

-- different order of JSON Object items

-- presence of any formating whitespace (outside of a JSON String)

-- different formating of numbers (123.00 verus 123 or 1.23E2)

Note: If using the LabVIEW extension that uses NaN numbers, these are considered Equivalent, ignoring the ususal convention of NaN not equal to NaN.

Invalid JSON will always return "Not Equivalent" even if both inputs are identical.

This function looks only at the leading JSON Value in the input strings, ignoring any following characters.

Other JSON
