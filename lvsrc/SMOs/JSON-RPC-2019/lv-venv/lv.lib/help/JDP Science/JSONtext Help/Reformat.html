JSONtext.lvlib:Reformat.vi

Reformat a JSON Value, either packed with all unneeded whitespace removed, or "pretty printed" with extra whitespace and new lines to make the JSON more human readable. This will also serve to validate the JSON.

"Pretty Print Levels" is the number of deeper levels at which to apply pretty print. Use a high number to apply at all levels, and use zero to indicate no pretty print.

 JSON

Input JSON. Optional "Offset" indicates an offset in this string pointing to the start of the JSON item to operate on.

Offset

Optional Offset in the input "JSON" string. Default is zero, the start of the string. This Offset should point to the start of the JSON Value of interest (or to whitespace in front of it).

error in (no error)

Describes error conditions that occur before this node runs. This input provides standard error in functionality.

    		status

    		code

    		source

Pretty Print Levels

Nested levels into the JSON in which to use new lines and indentation to make the text more easily readable.

Note that short Objects/Arrays will not be so formated.

Alphabetically Sort Objects

Format empty string as null

If true, input strings that are empty (or only whitespace) are formatted as a null JSON value. Otherwise, an empty input string is an error.

Disable LabVIEW Extensions

Replaces all instances of LabVIEW JSON Extensions (NaN, Infinity and -Infinity) with null to ensure compatability with other JSON implimentations that do not support these.

Offset past Value

Offset, in the input JSON, just past the item of interest.

<JSON>First JSON Value

Output JSON. The <JSON> tag indicates to some downstream functions that this is JSON, rather than a standard LabVIEW string.

error out

Contains error information. This output provides standard error out functionality.

    		status

    		code

    		source

Rest of Input String

Rest of the input string, after the found JSON Value.
