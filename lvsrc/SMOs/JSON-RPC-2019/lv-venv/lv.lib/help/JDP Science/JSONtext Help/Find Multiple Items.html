JSONtext.lvlib:Find Multiple Items.vi

Find the JSON items at the JSON Path specified and returns them as an array.

Paths specify an item in a tree of nested Elements, and start with $. Each element is either in the form .name, for Object items, or [n] for Array index ([*] for all elements). Multiple Object items can be specified by the list form .[abc,def].

If the Path input does not start with a $, then it is interpreted as a single Object Item name, rather than a path.

Example JSON:

{"A":[{"C":123,"D":4},{"C":567,"D":8}],"B":{"F":9},"E":true}

At path $.A[*].C are 123 and 567

At path $.B (or just B) is {"F":9}

At path $.B.F is 9

At path $.[B,E] are {"F":9} and true

 JSON

Input JSON. Optional "Offset" indicates an offset in this string pointing to the start of the JSON item to operate on.

$Path or Item Name

JSON Path notation starts with a $. If not starting with a $, this input is treated as an item name (so <name> and $.<name> are equivalent).

Offset

Optional Offset in the input "JSON" string. Default is zero, the start of the string. This Offset should point to the start of the JSON Value of interest (or to whitespace in front of it).
