JSONtext.lvlib:Merge Cluster into Object.vim

Merges a LabVIEW Cluster into a JSON Object.

Merging means that each element of the Cluster is converted to JSON and inserted into the object. Insertion is at the root level unless a Path to a subObject is specified.

Note that this merging is not recursive into subObjects.

 JSON

Input JSON. Optional "Offset" indicates an offset in this string pointing to the start of the JSON item to operate on.

Object $Path or Item Name

JSON Path notation starts with a $. If not starting with a $, this input is treated as an item name (so <name> and $.<name> are equivalent).

Any Cluster

This malleable VI will accept any Cluster.

    		element

    Placeholder: the cluster can contain any number of any type of element.

Offset

Optional Offset in the input "JSON" string. Default is zero, the start of the string. This Offset should point to the start of the JSON Value of interest (or to whitespace in front of it).

error in (no error)

Describes error conditions that occur before this node runs. This input provides standard error in functionality.

    		status

    		code

    		source

enable LabVIEW extensions? (T)

Enables LabVIEW JSON extensions to support NaN and Inf values of floating-point numbers. Not all JSON parsers support these extensions.

By default, this input is true, to match the inbuilt LabVIEW JSON functions; however, one may enforce no use of of the LabVIEW Extension using the Project Conditional Disable Symbol "JSONtext_Strict" = "True".

LVClass Serializer

Optional way to specify a JSON format for LabVIEW Objects. Provide a Child Class at this input that impliments LVObject<-->JSON conversion.

If no Serializer is provided, then LVObjects are serialized by the default method (flattened object in a JSON string).

<JSON>Merged

Output JSON. The <JSON> tag indicates to some downstream functions that this is JSON, rather than a standard LabVIEW string.

error out

Contains error information. This output provides standard error out functionality.

    		status

    		code

    		source
